# _Aleph 笔记知识库

一个基于 Typora + Git 的个人知识管理（PKM）库，遵循「输入 → 加工 → 输出」三层结构

## 二、笔记分类体系

### 2.1 输入层（他人内容）

- `book-@`：读书笔记——「模型提取 + 概念框架」（作者的观点）
- `ref-`：外部参考——文章 / 视频 / 资料的方法与观点

### 2.2 加工层（合成产物）

- `toolkit-@`：多个模型/概念合成的**可用新工具**——能在具体场景"想用就拿出来用"
- `card-@`：单一概念的稳定结晶——知识的最小单元；含核心定义 + 一张可视化或对比 + ≥2 个 book/ref 链接

`toolkit-@` 与 `card-@` 的差别：toolkit 是**多概念合成**（如"贝叶斯+斯多葛+反思 = 行动前/中/后三层决策"）；card 是**单概念结晶**（如"精度操控三型：锁死/通胀/坍塌"）。

### 2.3 过程层（持续追踪）

- `tracking-@`：行动 / 状态 / 实践追踪——持续更新，**不依附某本书**（按书做容器是反模式，已废弃）
- `movie-@`：观影笔记

### 2.4 自我认知层（开放问题）

- `atom-`：当前真实存在的认知摩擦、问题或困惑
- `hypo-`：基于一个或多个 `atom-` 形成的阶段性假设 / 解释 / 猜想（预留插槽，目前 0 用）

两点说明：

- `atom-` 与 `hypo-` 不是线性升级关系，而是**循环关系**：`atom- → hypo- → 新的 atom- → 修正 / 替换 hypo-`
- `hypo-` 只是暂时压缩问题的工作模型，**绝不代表这个问题已经得到解决**

### 2.5 导航层（不是笔记，是地图）

- `moc-@`：问题导航地图（必须有驱动问题，参见 §三）
- `!moc-Overview.md`：知识库总索引（cluster + moc 全景）

## 三、cluster 与 moc

笔记的两种组织方式，**不在同一个轴上**：

- **cluster（簇）** = 静态聚类，回答"这些笔记讲的是同一类东西"。例：所有沟通类笔记是一个 cluster。cluster **没有独立文件**，只在 `!moc-Overview.md` 中以章节形式存在。
- **moc（Map of Content）** = 问题导航地图，回答"我从哪里进、走到哪里去"。一张 moc 必须有一个**驱动它存在的问题**。

判断要不要建 moc：

- ✅ 有一个反复出现、跨多个 cluster 的问题——例：`moc-@认知链路` 串了贝叶斯 / 自由能 / 多巴胺 / 基因，回答"人是怎么决策的"
- ❌ 只是把同主题笔记列出来——这是 cluster，不需要 moc 文件
- ❌ 状态追踪表（如 `moc-@健康` 的最后更新表）——这是 dashboard，不是导航

**默认姿势**：先有问题，再造 moc。cluster 只是物理归簇；moc 是被问题召唤出来的路径。

### `moc-@` vs `toolkit-@` 边界

容易混淆，差别抓一句话：

- `moc-@` 只**链接**——内容是"X 笔记讨论 Y" + 链接 + 上下文，没有新合成
- `toolkit-@` 是**合成产物**——把 N 个模型组合成 1 个可用的新工具流程

如果你写的内容**没有新合成**，只是串联和导航——那是 moc。如果你写的内容**有可使用的新工具流程**——那是 toolkit。

## 四、统一字段结构

每条笔记建议包含以下字段：

- `title`：标题
- `type`：book / ref / tracking / toolkit / card / moc / movie / atom / hypo
- `coreConcepts`：本笔记的核心模型或关键词列表
- `source`：原始来源或书名 / 作者（toolkit / card 类填"提炼自哪条 tracking 或 book"）
- `appliedModels`：tracking 列出应用的模型；book/ref 列出可应用的模型
- `actionItems`：可直接落地的行动或实验
- `links`：关联笔记，如 `book-@...` / `ref-...` / `toolkit-@...` / `card-@...` / `tracking-@...`
- `comment`：当前状态、疑问、补充说明

## 五、层级标签定义

### 5.1 认知层

用于描述「思维结构」、「认知模型」、「决策机制」等。

- 示例：系统 1/2、贝叶斯、自由能、认知偏差、前景理论

### 5.2 行动层

用于描述「执行方法」、「实践路径」、「行动设计」等。

- 示例：刻意练习、MVP、环境设计、决策最小化、SQ3R

### 5.3 系统层

用于描述「体系构建」、「工作流设计」、「长期反馈循环」等。

- 示例：DKIA、输入/处理/输出、认知负荷、知识库结构、工具链
