# 工具概述

把 [card-@精度操控三型](card-@精度操控三型.md) 这种**描述性原则**转成**决策点的触发式 checklist**。

原因：原则只写在笔记里会在情绪起来时被绕过。2026-05-02 的真实案例——我在 tracking-@PKM ##2026-05-01 §六 刚自我提醒"不要再陷入鼓捣笔记软件陷阱"，**同一篇笔记里就开始认真考虑加 Notion**。这证明：

> 知道一条原则、和真正在决策点调用它，是两件事。

要解决的问题不是"再多写一遍原则"，而是把原则**落到具体动作的强制检查**——这才是 toolkit（多概念合成的可用流程）应当承担的角色。

---

# 触发条件

当我开始考虑下面任何一种动作时，**必须先答完下面 5 问再决策**：

- 加一个新软件 / 新平台
- 启用一个新插件
- 接入一个新服务
- 折腾现有工具的高级配置（用法之外的）

---

# 5 问 checklist

## 1. 它解决的是哪个**具体真问题**？

不接受这种回答：

- "可视化更好"
- "更方便"
- "我看别人在用"
- "可能以后会有用"

只接受能指向**具体一条 tracking** 或**某次具体卡顿**的回答，例如：

- "tracking-@健康-饮食结构 已有 23 条记录，无法快速看趋势——这是表格视图能解决的具体卡顿。"

如果第 1 问答不出来，**直接拒绝**，写进 `_inbox.md` 等 1 周再回看。

## 2. 这个问题用现有工具**完全不能**解决吗？

不接受"不够好用"。只接受"完全没法做"。

如果现有工具能做但我没做，先问：**为什么至今没做？** 大概率是因为我不想花精力，那么换工具也只是把同一种"不想花精力"换个新形式延续。

## 3. 加进来后我每周需要**多少分钟**维护？

具体数字。不能写"不多"。如果答不出来，说明**还没真正想过维护成本**——这是精度通胀的最经典信号。

参考阈值：

- < 10 分钟/周：可以考虑
- 10-30 分钟/周：必须有强力理由
- > 30 分钟/周：默认拒绝

## 4. 三个月后我会**后悔**吗？

凭直觉打 Y / N。

如果**犹豫**——不要硬决定。先把这个想法扔进 `_inbox.md`，**等满 1 周**再回看。一周后还想加，再考虑下一步；一周后不想了，原本就是冲动。

## 5. 这个想法是不是就在我刚提醒过自己 [card-@精度操控三型](card-@精度操控三型.md) 之后冒出来的？

如果是——**默认拒绝**。

理由：刚提醒就违反，是精度通胀复发的最强信号。这种时候情绪已经在主导，决策不可信。强制冷却。

---

# 使用边界

**适用**：

- 加工具（新软件 / 新插件 / 新服务）
- 折腾配置（超出基本用法的折腾）

**不适用**（这些有各自的判定逻辑）：

- 笔记内容创作 → 走 README §3.1 决策树
- 笔记结构调整（cluster / moc / card） → 走 README §3 + note-intake.md
- 删工具 → 自由处理（删比加便宜）
- 学新概念 / 新模型 → 不在工具范畴

# 复现案例

**2026-05-02**：tracking-@PKM ##2026-05-01 §六 写下"不要再陷入鼓捣笔记软件陷阱"，**同一篇笔记里**开始考虑加 Notion 做多维表格 + 看板。Claude 反诊断时点出复发，触发了本 toolkit 的设计。

这是该 toolkit 被设计出来的**原始触发场景**——也是它未来生效的真实标准：下次再有类似冲动，**实际跑一遍 5 问，看是否真能拦住**。

---

# 二阶提醒

这条 toolkit 本身也是一个工具。如果未来发现它**没拦住**任何冲动（5 问被简单跳过、或者答案被自我合理化），那它就不是有效的钩子，需要换形式（例如挪到 `_inbox.md` 顶部做硬常驻 / 改成对话开始时的强制问句 / 或者作废）。

> 工具自己也要被这套 checklist 检验。
