### 笔记前缀划分

我的笔记分为四个前缀：

- `book-` / `ref-`：他人的观点与结构化表达
- `atom-`：我当前真实存在的认知摩擦、问题或困惑
- `hypo-`：基于一个或多个 `atom-` 形成的阶段性假设 / 解释 / 猜想

要说明两点：

- `atom-` 与 `hypo-` 不是线性升级关系，而是**循环关系**： `atom- → hypo- → 新的 atom- → 修正 / 替换 hypo-`

- `hypo-` 只是暂时压缩问题的工作模型，**绝不代表这个问题已经得到解决**

------

### 元数据设计目标（仅针对 atom- / hypo-）

做笔记是为了记录和管理认知摩擦，因此元数据只描述两件事：

- 我是否正在为这条笔记投入注意力
- 我对这条笔记的理解是否仍然存在明显摩擦

为避免中间态导致的自我安慰，全部采用二元化表达：

```
friction_level: high | low
focus: on | off
```

------

### 元数据含义的系统性定义

- `friction_level: high`
   我在思考或复述时会卡顿、回避， 往往需要借助外力（如 AI、重读材料）才能继续
- `friction_level: low`
   我可以相对顺畅地独立复述、使用或解释该内容， 不依赖即时外部提示
- `focus: on`
   我当前正在为此投入认知精力
- `focus: off`
   我主动选择暂时不为此投入认知精力（不等同于已解决）

------

### 防止心理暗示的约束规则

- 对于所有 `focus: off` 的笔记，
   必须**在开头写一句话，说明我当前不继续投入精力的理由**

------

### hypo- 的额外约束（防止假设硬化）

每一条 `hypo-` 笔记必须包含以下两点：

1. **来源说明**：

   ```
   derived_from:
     - atom-xxx
     - atom-yyy
   ```

   用于明确该假设源自哪些具体问题

2. **我目前最不确定的地方**（自由文本）：
    明确记录该假设尚不稳固、可能失效或尚未想通的部分
