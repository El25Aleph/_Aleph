# 笔记知识库 Schema 定义

这是你与我后续对话及笔记管理的结构性范式定义文件。它定义了笔记分类、核心字段、更新流程和对话模式。

## 1. 核心分类

- `book`：读书笔记，主要是“模型提取 + 概念框架”。
- `ref`：外部参考，总结文章/视频/资料中的方法与观点。
- `tracking`：行动记录、反思、模型迁移与实践反馈。
- `toolkit`：多个模型或概念的桥接、综合与工具化。

## 2. 统一字段结构

每条笔记建议包含以下字段：

- `title`：标题
- `type`：book / ref / tracking / toolkit
- `coreConcepts`：本笔记的核心模型或关键词列表
- `source`：原始来源或书名/作者
- `appliedModels`：如果是 tracking，则列出应用的模型；如果是 book/ref，则列出可应用的模型
- `actionItems`：可直接落地的行动或实验
- `links`：关联笔记或桥接文档，如 `book-@...` / `ref-...` / `toolkit-@...` / `tracking-@...`
- `comment`：当前状态、疑问、补充说明

## 3. 层级标签定义

### 3.1 认知层

用于描述“思维结构”、“认知模型”、“决策机制”等。

- 示例：系统 1/2、贝叶斯、自由能、认知偏差、前景理论

### 3.2 行动层

用于描述“执行方法”、“实践路径”、“行动设计”等。

- 示例：刻意练习、MVP、环境设计、决策最小化、SQ3R

### 3.3 系统层

用于描述“体系构建”、“工作流设计”、“长期反馈循环”等。

- 示例：DKIA、输入/处理/输出、认知负荷、知识库结构、工具链

## 4. 新笔记对比流程

### 4.1 新笔记进入时

1. 判断笔记类型：`book` / `ref` / `tracking` / `toolkit`
2. 抽取 `coreConcepts`
3. 对照 `docs/_wiki.md` 的现有模型列表
4. 标记：
   - `existing`：已存在于 wiki 中
   - `new`：全新模型或新的组合方式
5. 如果是 `tracking`，确认是否有“模型应用记录”
6. 如果是 `book/ref`，确认是否应加入“潜在落地场景”

### 4.2 对比判定标准

- 概念是否属于已知模型？
- 是否存在新的概念组合？
- 是否引入新的实践流程？
- 是否补充了已有模型的应用场景？
- 是否提供了新的跨领域连接？

## 5. 对话范式

以后你和我讨论笔记时，可以使用以下结构化问题模板：

### 5.1 发现模型

- 这篇笔记的 `coreConcepts` 是什么？
- 它在哪个层次：认知层 / 行动层 / 系统层？
- 它是否新增模型，还是对现有模型进行扩展？

### 5.2 比较与链接

- 它与现有 `docs/_wiki.md` 中的哪些模型相关？
- 是否应补充到 `tracking` 的应用场景？
- 是否应补充到 `toolkit` 的桥接逻辑？
- 是否需要更新 `docs/_wiki.md` 的“持续更新指南”？

### 5.3 迭代与输出

- 这篇笔记下一步可以做什么行动？
- 是否能生成一个简单的“落地实验”或“复盘模板”？
- 是否需要把它写成一条 `tracking` 日志？

## 6. 常用模型对照表

| 模型类别 | 常见模型 | 说明 |
| --- | --- | --- |
| 认知层 | 系统 1 / 2 | 决策机制与认知负荷 |
| 认知层 | 贝叶斯 / 自由能 | 预测与更新模型 |

## 7. 文件角色说明

- `docs/_wiki.md`：你的知识库核心概览与最重要的对比基准。
- `docs/_schema.md`：你的笔记类型、字段与对话格式的标准定义。

## 8. 版本与维护

- 2026-04-17：创建初版。
- 每次新增 `book/ref/tracking/toolkit` 时，先按 `docs/_schema.md` 提取信息，再按 `docs/_wiki.md` 进行对比与更新。
- 未来如果你希望，可以继续把该 schema 扩展为“文件头模板”或“笔记元数据标准”。
