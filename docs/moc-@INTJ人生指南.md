# §0 这张 moc 怎么读

**它不是又一张工具地图，而是知识库里第一份"用户手册"。**

现有 6 张 moc 的主语是"事"——怎么决策、怎么沟通、怎么思考。这张 moc 的主语是"我"——这台 INTJ 人格机器，在哪个发展阶段、该调用哪些工具、容易卡在哪些坑里。两层叠在一起，知识库从"工具仓库"变成"工具仓库 + 操作员手册"。

**真正的驱动问题**（作者本人 2026-06-25 定调）：

> 我无法改变他人和社会/系统/环境，但可以改变我自己。有了这篇 moc，可以更好地剖析我自己。这个知识库所有的笔记，都应该服务于这个目标。

这恰好对应斯多葛主义的控制圈 / 关注圈二分——关注圈（他人/系统/环境）可以理解，但不消耗精力去改变；控制圈（自己）才是真实可行的工作场。

底层引用：[ref-INTJ人生指南](ref-INTJ人生指南.md) 是 B 站 up 主"小白价值投资"的二手总结 + 作者批注，本 moc 在它基础上做"串联地图"。

## §0.1 八维映射的元规则（重要）

**八维不是硬归类，是动作标签。**

判定一篇笔记"属于哪一维"的标准——**不是看笔记内容是什么，而是看读这篇笔记时大脑在做什么动作**：

- 同一本《置身事内》：在做"中央/地方/财政"结构提取 → Te；在做"央地博弈 ↔ 电力电子稳定性"横向联想 → Ne
- 同一本《学会提问》：在做论证结构精化 → Ti；在做"如何让对方说出真实需求" → Fe
- 同一篇 tracking-@健康-饮食结构：在查"哪些食物会让我反应" → Si；在写"今天吃完这碗饭身体感觉如何" → Se

所以 §2 下面那些归类都是**"主要承载哪一维"**——不是排他性硬归类。同一篇笔记可以在不同场合被读出不同维度。这条元规则也意味着：**Ne 维"看起来薄"不一定是真薄，可能是同样的笔记在 Te / Ti 动作下被消化了，没机会以 Ne 形态留下痕迹**。

# §1 四阶段发展线

来源：[ref-INTJ人生指南](ref-INTJ人生指南.md) §INTJ 人生四大阶段

不是四个并列的能力，而是一条**发展次序**——前一阶段是后一阶段的基础。

| 阶段 | 功能链 | 状态特征 | 此阶段应重点调用 |
|---|---|---|---|
| 1 | Ni-Ti 循环 | 沉迷理论 / 计划 / 抽象，眼高手低 | [book-@思维的框架-I](book-@思维的框架-I.md) 提供 Ti 框架；但**需要打破**：用 [book-@刻意练习](book-@刻意练习.md) 强行启动 Te |
| 2 | Ni-Te 循环 | 有所小成（考试 / 工作）| [ref-5am-project](ref-5am-project.md) / [book-@起床后的黄金一小时](book-@起床后的黄金一小时.md) / [ref-战略性懒惰](ref-战略性懒惰.md) |
| 3 | + Fi 筛选 | 明确"不追求什么"，聚焦真正热爱 | 文学回响 cluster（Fi 的训练场）+ 5-25 法则 + 内部积分卡（均在 [ref-INTJ人生指南](ref-INTJ人生指南.md) §Fi 篇） |
| 4 | + Se 休息 | 懂得养精蓄锐 | [moc-@健康](moc-@健康.md) + 冥想 + 多元运动（见 [ref-INTJ人生指南](ref-INTJ人生指南.md) §Se 篇） |

**判断自己在哪一阶段的最低阈值**：
- 反复定计划而不执行 → 阶段 1
- 能做出成果但不知道为什么追求 → 阶段 2
- 知道追求什么但精力透支 → 阶段 3
- 以上都不卡 → 阶段 4 维护期

# §2 八维 ↔ 知识库映射

每一维的映射强度都标注 1-5（基于现有笔记的实际内容，不是直觉打分）。

## §2.0 八维各维到底是什么——我的理解校准（2026-06-29）

> ref 里 up 主怎么理解八维不重要，重要的是**我**怎么理解。2026-06-29 与 Claude 反茧房 check 我的八维定义，偏差如下。校准一律用"它能为我做什么"（Te 式）框定，不要"内倾直觉"那种 Ni 式抽象。
>
> ⚠️ **半成品**：下表「我最初的理解」列是 Claude 转述我 2026-06-29 的原话，**待我核对是否转述歪了**；「校准」列已填到位。

| 维 | 我最初的理解 | 校准（它能为我做什么）| 判 |
|---|---|---|---|
| **Ni** | 构建模型的直觉 | 把碎片收敛成可预测的模型 / 单一愿景 | ✓ 准 |
| **Te** | 行动力 / 执行力 | 把目标拆成**自动化的系统 / 环境**（减少决策摩擦）；是引擎，不是"用力做"——油门 ≠ 方向盘 | 偏 |
| **Fi** | 筛选机制 | 定义"什么对我而言**值得**" = 我的奖励函数（判据 = 价值 / 真实的我）| 差判据 |
| **Se** | 对外界的感知 / 当下体验 | 让我真"在场"（读身体信号 / 应对突发 / 享受当下）；已有意放在 PKM 之外的手账本 | ✓ 准 |
| **Ne** | 搜集和自己不相关的信息（≈ PARA 的 area/resource）| **主动跨领域蹦连接 / 生 possibility**——不是被动收集，更不是存储桶（拿"放哪"比"怎么想"是类目错位）| 偏 |
| **Ti** | 盘逻辑的能力 | 把一个想法的内部逻辑盘到自洽、抠细节；与 Te 配——Te 给粗框架、Ti 抠细（ref 的 Ni-Te-Ti 三步）| ✓ 准 |
| **Fe** | 和人沟通的能力 | 读懂房间的情绪 + 感知对方真正的需要 + 调和关系；沟通是输出，**情绪雷达**才是功能；对我是刻意戴的社交面具 | 差判据 |
| **Si** | 对细节的把控能力 | 存档过去的具体经验 + 拿现在比过去（核心是**时间 / 记忆**，不是"细节"）；我最弱的第八功能，已外包给 tracking + checklist | 偏 |

**这次校准真正的收获——不是 8 个定义，是一个 pattern**：

> 我**按"输出"定义功能、漏掉"按什么"那层判据**（Fi 按什么筛 = 价值 / Fe 凭什么沟通 = 情绪 / Te 靠什么行动 = 系统）。这是"Te 式提问"自带的盲点——**Te 盯结果，天然不爱问"凭什么"**。
>
> **解药：定义 / 分类任何东西之前，多问一句"按什么？"** 与 §3「fancy 形式陷阱」、以及"合并相邻层"的倾向同源（都是跳过判据 / 区分层）。

注意：这条 Se 通路**不在知识库的设计范围内**，moc 这里只挂占位标签，不试图把它数字化、不在 audit 中说它"薄"、不主动建议把回忆笔记化。

# §3.5 元层风险：moc 本身可能成为 Ni 模型坍缩源

> 2026-06-27 作者论证"Ni 模型坍缩"反 pattern 后明确认可的潜在风险——挂起未触发的 watcher，避免坍缩期到来时识别不出来。

这张 moc 本身也是一个 Ni 模型。**任何 Ni 模型都可能从"工具" 演化成"牢笼"**：

| 阶段 | 状态 | 体征 |
|---|---|---|
| **工具期**（当前）| moc 是脚手架 + 多巴胺催化剂，让八维与现有笔记产生命名 + 关联 | 用 moc 后行为更清晰、识别陷阱更快、行动更稳 |
| **坍缩期**（潜在风险）| moc 变成认知牢笼——所有自我观察都被八维框架预先过滤 | 用 moc 后反而觉得自己更迷茫 / 更不像自己 / 在为 moc 服务而不是 moc 服务自己 |

## 触发信号（显式写出避免遗漏）

- 自我观察总是 immediately 套到八维某一维上，看不到框架外的东西
- 阅读新书 / 看新人时本能先问"这属于哪一维"，再决定是否消化
- moc 的术语开始替代具体经验——说"我陷入 Ni-Ti 循环了"比说"我又卡住了" 更顺嘴 → 抽象正在覆盖具体
- moc 看上去解释了所有事 → 元 Fi 警觉信号（**解释一切 ≈ 解释不了任何具体的**）

## 解药

作者本人 2026-06-27 提出的**"模型重构的勇气"** —— Te 自我授权的更深应用，敢于推翻已建模型（详见 [_inbox.md] 2026-06-26 元 Fi 讨论 + memory `user_three_function_self_model.md`）。

## 应用规则

- 不要为了让 moc 完整而强行套用它到不合适的场景
- 触发信号出现时，先用 Te 在数字 PKM 之外做一段"无 moc 期"实验（比如一两周纸质本子记录不套 moc 语言）
- 这张 moc 不是 INTJ 人生指南的最终版本，是**当前版本**——版本会变

## 为什么现在显式写出

作者 2026-06-27 论证 moc-INTJ 作为"多巴胺催化剂"时，刚好暴露这条潜在风险——**任何催化剂用过头会变成抑制剂**。这条风险**现在不需要应对**（moc 刚建好还没用过），但 watcher 必须在场。挂起在此，等触发信号出现时再用。
