## 五、维护节奏

- 新增笔记 → 选 cluster → 在第三节对应 H2 节加链接，第一节篇数 +1。
- 新建 moc → 在第二节加一行 → 同时更新对应 cluster 行的"关联 moc"列。
- 半年回看一次 cluster 表：长期"待挖"且未行动的 cluster 是否该 archive 到 `docs/archive/`。
- 待挖问题真的开始挖矿且产出洞见 → 按 [README §3](../README.md) 判定是否折出独立 moc 文件。
