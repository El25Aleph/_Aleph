# 知识库 Wiki 主页

> **总索引**：[!moc-Overview](!moc-Overview.md) ——12 个 cluster + 6 张 moc 的入口表

这是你的笔记知识库的“活页手册”，它不是一个静态总结，而是一个持续对比、迭代、补充的核心节点。

## 目的

- 汇总当前 `book-` / `ref-` 笔记中的重要模型与概念。
- 记录这些模型如何在 `tracking-` 中已被应用。
- 链接 `toolkit-` 中的桥接思路，作为“模型到实践”的中继。
- 作为新笔记到来时的对比基准：新增概念、更新模型、补齐应用场景。

## 结构说明

本文档分为四个核心部分：

1. 核心模型与概念
2. Tracking 中的落地与应用

## 4. 持续更新指南

### 4.1 新笔记进来时的对比流程

1. 读取新笔记标题与类型：`book` / `ref` / `tracking` / `toolkit`
2. 提取该笔记的核心概念与模型
3. 对照本 wiki 的“核心模型与概念”列表：
   - 如果是已有模型，则补入具体应用场景
   - 如果是新模型，则加入“新增概念”区，并标注来源
4. 检查是否有新的实践方式需要补到“Tracking 的落地与应用”
5. 检查是否有新的桥接方法需要补到“Toolkit 的桥接与链接”
6. 以本文件为核心节点，写出一句“这篇新笔记在本体系中的定位”

### 4.2 更新内容格式建议

每次补充时，建议保持下面三层结构：

- `核心概念`：这篇笔记带来了什么模型、概念或术语？
- `应用场景`：这个模型当前在哪些 tracking / toolkit 中已经出现或可以补充？
- `链接节点`：它与哪些现有笔记直接关联？（例如 `book-@二八法则.md` / `ref-战略性懒惰.md` / `toolkit-@认知链路.md`）

### 4.3 要持续记录的几个关键维度

- 模型名 / 关键词
- 该模型的来源类型（book/ref/toolkit）
- 已落地实践（tracking 文件名）
- 预期可落地场景
- 是否属于“认知层”、“行动层”或“系统层”

## 5. 当前注意点

- `book` 与 `ref` 更偏“模型和概念输入”；

这三个层次之间的持续对比，是本 wiki 的核心价值。
