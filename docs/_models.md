# 模型清单（_models）

> **总索引**：[_overview](_overview.md) ——12 个 cluster + 6 张 moc 的入口表
>
> **关于本文性质**：这里记录的是当前理解的模型与概念，不是权威结论。任何条目都可能被新证据修正——文件名从 `_wiki.md` 改为 `_models.md` 就是为了去掉"百科"的语义重量。

这是知识库的活页手册，持续对比、迭代、补充。两件事：

1. **§1 模型清单**——当前 `book-` / `ref-` 中已识别的模型与概念，按 4 类分组。新笔记进来时拿这里做对照基准，判定是否新增 / 扩展 / 补充应用场景。

## 2. Tracking 中的落地与应用

### 2.1 已经应用的模型

- `刻意练习 + MVP`：明确区分“偏理论知识”和“偏实践知识”，分别用 `费曼学习法` 与 `刻意练习 + MVP` 落地。
  - 典型文件：`tracking-@PKM-2024.md`

- `MECE`：作为个人知识库设计的拆解工具，用于分类、标签与检索结构。
  - 典型文件：`tracking-@PKM-2025.md`

- `认知负荷`：用于比较纸质记录与笔记工具，帮助判断笔记形式的便捷性与可操作性。
  - 典型文件：`tracking-@PKM-纸质记录.md`

- `DKIA 模型`：Drive / Knowledge / Integration / Action，用作个人系统设计框架。
  - 典型文件：`tracking-@PKM-2025.md`

### 2.2 还可以继续落地的模型

- `贝叶斯定理`：决策日志中记录“先验 / 新证据 / 后验”更新过程。
- `系统 1 / 系统 2`：行动复盘中记录“本次决策更多依赖哪个系统”。
- `战略性懒惰`：记录环境设计实验、模板/默认值/自动化流程、决策最小化结果。
- `SQ3R`：阅读工作流中写入“Survey / Question / Read / Recite / Review”节点。
- `设计思维`：项目跟踪中加入“观察→洞察→原型→测试→迭代”流程。
