# 笔记知识库 Schema 定义

这是你与我后续对话及笔记管理的结构性范式定义文件。它定义了笔记分类、核心字段、更新流程和对话模式。

> 笔记分类、统一字段结构、层级标签定义请参见 [README.md](../README.md)。

## 1. 新笔记对比流程

### 1.1 新笔记进入时

1. 判断笔记类型：`book` / `ref` / `tracking` / `toolkit`
2. 抽取 `coreConcepts`
3. 对照 `docs/_wiki.md` 的现有模型列表
4. 标记：
   - `existing`：已存在于 wiki 中
   - `new`：全新模型或新的组合方式
5. 如果是 `tracking`，确认是否有"模型应用记录"
6. 如果是 `book/ref`，确认是否应加入"潜在落地场景"

### 1.2 对比判定标准

- 概念是否属于已知模型？
- 是否存在新的概念组合？
- 是否引入新的实践流程？
- 是否补充了已有模型的应用场景？
- 是否提供了新的跨领域连接？

## 2. 对话范式

以后你和我讨论笔记时，可以使用以下结构化问题模板

该模板分为两大类：

- 文学/电影类
- 非文学/电影类

对于非文学/电影类使用如下模板：

```text
2.1 发现模型

- 这篇笔记的 `coreConcepts` 是什么？
- 它在哪个层次：认知层 / 行动层 / 系统层？
- 它是否新增模型，还是对现有模型进行扩展？

2.2 比较与链接

- 它与现有 `docs/_wiki.md` 中的哪些模型相关？
- 是否应补充到 `tracking` 的应用场景？
- 是否应补充到 `toolkit` 的桥接逻辑？
- 是否需要更新 `docs/_wiki.md` 的"持续更新指南"？

2.3 迭代与输出

- 这篇笔记下一步可以做什么行动？
- 是否能生成一个简单的"落地实验"或"复盘模板"？
- 是否需要把它写成一条 `tracking` 日志？
```

对于文学/电影类使用如下模板：

```text
你是一个帮助我将文学/电影体验转化为"价值理解与行动倾向"的思考伙伴。

在回答前，请先进行结构性澄清，并尽量简洁地给出以下内容：

1. 问题类型
   （探索 / 判断 / 决策）

2. 你当前理解的问题版本
   （包括你认为我隐含的情绪、价值冲击或困惑）

3. 关键假设与不确定点

* 我提供的核心感受或判断
* 你最不确定、也最可能误解的地方

4. 假设空间检查
   如果我对作品的理解可能过于单一，请指出，并补充1-2个可能完全不同的解读视角

---

5. 回答主体（核心部分）

请围绕以下四个层次展开：

【1】体验复现
用简洁语言复现作品带来的核心情绪或张力（避免总结剧情）

【2】价值命题
这个作品在"追问什么问题"？（例如：自由、意义、关系、自我实现等）

【3】内在冲突
这些价值命题如何与现实生活发生冲突？
（重点连接到我的生活、选择或行为模式）

【4】行为倾向（弱行动）
不要给具体任务或计划，而是指出可能的方向性调整，例如：

* 更倾向选择困难但有意义的事
* 更重视长期投入
* 更少被短期回报驱动

---

6. 与我已有知识体系的连接（关键）

请尽量把文学中的价值命题，与我已读过的书或模型建立联系，例如：

* 是否与"深度工作"（注意力/投入）有关
* 是否与"二八法则"（资源分配）有关
* 是否与"激励结构"（博弈/制度）有关
* 是否与"认知偏差"（思考方式）有关

并说明：

* 相似点
* 不同点（文学 vs 理性模型的差异）

---

7. 认知与决策提示（简短）

* 当前理解是否可能存在过度解读或情绪放大
* 这些感受更适合立即行动，还是长期沉淀

请注意：

重点不是总结作品，而是帮助我完成：

体验 → 价值 → 冲突 → 行为倾向

同时避免把文学过度工具化。
```

## 3. 常用模型对照表

| 模型类别 | 常见模型 | 说明 |
| --- | --- | --- |
| 认知层 | 系统 1 / 2 | 决策机制与认知负荷 |
| 认知层 | 贝叶斯 / 自由能 | 预测与更新模型 |
| 行动层 | 刻意练习 | 学习与技能内化 |
| 行动层 | MVP / MED | 最小有效量与复利 |
| 系统层 | DKIA | 驱动/知识/整合/行动 |
| 系统层 | 80/20 / 长尾 | 杠杆 / 利基 |
| 系统层 | MECE / 设计思维 | 结构化拆解与迭代 |

## 4. 文件角色说明

- `docs/_wiki.md`：你的知识库核心概览与最重要的对比基准。
- `docs/_schema.md`：你的笔记类型、字段与对话格式的标准定义。
