# 笔记检索（note-search）

## 何时启用

用户的请求涉及**回忆库内已有内容**，而不是产生新内容。典型触发：

- "我之前在哪里写过……"
- "关于 X 的笔记都有哪些"
- "找一下提到 Y 的地方"
- "把所有 atom-/hypo-/tracking- 列出来"

## 检索策略（按顺序尝试，命中即停）

### 第 1 步：识别检索维度

先判断用户在问哪一类，因为 _Aleph 的笔记是**强类型**的：

| 前缀 | 含义 | 用户意图举例 |
| --- | --- | --- |
| `book-@` | 读书笔记（模型提取） | "讲过 80/20 的书" |
| `ref-` | 外部参考（文章/视频摘要） | "关于卡片笔记法的资料" |
| `tracking-@` | 行动记录与反思 | "我对君主论的实践" |
| `toolkit-@` | 多模型桥接 | "把贝叶斯和自由能串起来的笔记" |
| `atom-` | 当前认知摩擦/困惑 | "我现在还没想清楚的问题" |
| `hypo-` | 阶段性假设 | "我对 X 的临时解释" |
| `moc-@` | 主题地图（Map of Content） | "认知链路、沟通、理性思考的索引" |
| `movie-@` | 观影笔记 | |

### 第 2 步：选用合适的检索工具

- **按文件名/前缀找** → 用 Glob，例如 `docs/book-@*认知*.md`、`docs/atom-*.md`
- **按内容关键词找** → 用 Grep，记得：
  - 默认中文关键词命中率高，**直接搜中文词**而不是英文译名
  - 对人名/书名加引号边界："理查德·科克"、"《置身事内》"
  - 用 `-l` 模式先列出文件，再读最相关的几篇
- **跨多个维度交叉** → 先用 Grep 缩小到一组文件，再用 Read 取上下文
- **moc-@ 是地图层**：当用户问"X 这个主题下有哪些笔记"时，**先看 moc-@X.md 是否存在**，再看里面已经索引过哪些链接

### 第 3 步：回答时给出可点击路径

按照本仓库的 IDE 约定，路径必须用 markdown 链接，例如：

> 在 [book-@思考，快与慢](docs/book-@思考，快与慢.md) 第 3 节、[ref-人性矩阵系列-06-认知](docs/ref-人性矩阵系列-06-认知.md) 都讲过双系统。

不要使用反引号包路径，不要给绝对路径。

## 反模式

- **不要直接 Read 整个 docs/ 目录**：先用 Grep/Glob 缩小，再读具体文件。
- **不要凭训练数据回答**："你笔记里应该有 X" 这种猜测是禁的，要么找到具体文件，要么明说"没找到"。
- **不要混淆 atom 与 hypo**：atom 是问题，hypo 是临时答案。用户问"我的困惑"不要返回 hypo。
- **不要忽略 archive/**：旧的 tracking 在 `docs/archive/` 下，按年归档（如 tracking-@PKM-2024.md），用户问"以前怎么记的"时要包含进来。

## 输出格式

检索结果按下列结构返回：

1. **直接命中**（≤3 篇最相关的）：每篇一行，markdown 链接 + 一句话摘要。
2. **次要相关**（可选，≤5 篇）：只给链接列表。
3. **没找到的话**：明确说"没有命中"，并提示用户用什么不同的关键词再试，**不要编造**。
