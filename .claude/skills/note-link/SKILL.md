# 笔记互联（note-link）

## 何时启用

- 用户写完一篇新 book-/ref-/atom-/hypo-，问「这和我以前的笔记有什么关系」。
- 用户说「补全 links 字段」「把这篇接到 moc 里」。
- 主动时机：当 note-writing 写完一篇正文后，**立即跑一次 note-link 给出候选链接**，由用户决定是否采纳。

## 链接的四种类型

不要把所有关联都写成一个混在一起的链表，应该分类：

| 关系 | 含义 | 该写在哪 |
| --- | --- | --- |
| **同源链接** | 同一作者/同一概念簇的笔记 | `links` 字段 |
| **桥接链接** | 跨领域但底层模型相同（如自由能 ↔ 贝叶斯 ↔ 双系统） | 写进 `toolkit-@` 或 `moc-@` |
| **反例链接** | 立场相反或结论冲突的笔记 | 在「批判性思考」段落显式标注 |
| **派生链接** | 这篇 hypo- 是从哪些 atom- 长出来的 / 这篇 atom- 引用了哪些 book- | atom 与 hypo 之间用 `parentRef` / `childCards` 字段 |

## 操作流程

### 第 1 步：抽取候选关键词

从当前笔记的 `coreConcepts` 字段开始。如果字段为空或只有 1-2 个词，先读正文提取 3-5 个核心概念。

### 第 2 步：用 Grep 跨库查找

对每个关键词跑一次 Grep（中文为主）：

```
Grep pattern="贝叶斯" path="docs/" output_mode="files_with_matches"
```

得到候选文件列表后，**按相关性而非数量挑选**——上限 5 篇，宁缺毋滥。

### 第 3 步：检查 moc-@ 是否需要更新

在 [docs/](docs/) 下找 `moc-@*.md`，看当前笔记的主题是否已被某个 moc 覆盖：

- 已覆盖 → 把当前笔记的链接加进对应 moc。
- 未覆盖但**已有 ≥3 篇**同主题笔记 → 提示用户「考虑新建 moc-@X.md」，但**不要替用户直接创建**。
- 单篇孤岛 → 不强行创建 moc，只在 `links` 里挂同源/桥接即可。

### 第 4 步：写回链接字段

在当前笔记里，找到（或创建）`links` 字段，按类型分组：

```
links:
  same: [ref-人性矩阵系列-06-认知, book-@思考，快与慢]
  bridge: [toolkit-@贝叶斯定理]
  counter: [ref-请不要神化双链笔记-少数派]
  parent: [atom-26-05]
```

如果笔记在仓库里被引用为目标，**还要去对方笔记里加反向链接**——单向链接是孤岛的开始。

## 反模式

- **不要为了凑数加链接**：5 篇相关 ≥ 20 篇沾边。
- **不要忽略 atom-/hypo- 的派生关系**：这是 _Aleph 体系最有价值的链接类型，参考 [README.md](README.md) 第 2.2 节"atom 与 hypo 是循环关系"。
- **不要重复 moc 已经做的事**：如果 moc-@认知链路 已经把贝叶斯、自由能、双系统串起来了，新笔记的 `links` 写一条「moc-@认知链路」就够了，不必把整条链路里的每篇都列一遍。
- **修改其他笔记前先确认**：要在另一篇笔记里加反向链接时，先 Read 看一下，避免破坏已有结构。

## 输出格式

不直接修改文件——**先列出建议清单**，等用户确认：

```
建议为 [当前笔记] 补充以下链接：

- 同源（2）：
  - [ref-人性矩阵系列-06-认知](docs/ref-人性矩阵系列-06-认知.md) —— 同样讲双系统
- 桥接（1）：
  - [toolkit-@贝叶斯定理](docs/toolkit-@贝叶斯定理.md) —— 把贝叶斯接到自由能
- 反例（0）
- 反向回链：建议在 [book-@思考，快与慢](docs/book-@思考，快与慢.md) 里加一条指向当前笔记的链接

是否采纳？
```
